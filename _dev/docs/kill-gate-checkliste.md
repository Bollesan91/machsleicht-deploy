# Kill-Gate-Checkliste — machsleicht.de

> Vor jeder neuen Seite diese 6 Fragen durchgehen. Nur wenn alle bestanden sind, wird die Seite erstellt.

## Gate 1: Themen-Fit
- [ ] Passt das Thema zum Kern von machsleicht? (Kindergeburtstag, Baby, Einschulung, Familien-Organisation)
- [ ] Würde eine Eltern-Zielgruppe danach googeln?
- Falls NEIN → Seite NICHT erstellen.

## Gate 2: Competitive Edge
- [ ] Was macht unsere Seite BESSER als die Top-3-Ergebnisse bei Google?
- [ ] Mindestens einer dieser Edges muss zutreffen:
  - **Tool-Edge:** Interaktives Tool (Planer, Generator, Rechner) das die Konkurrenz nicht hat
  - **Vollständigkeits-Edge:** Wir decken das Thema umfassender ab
  - **Praxis-Edge:** Konkretere Anleitungen mit echten Mengen, Zeiten, Kosten
  - **Struktur-Edge:** Bessere Entscheidungshilfe (z.B. nach Alter/Budget/Situation)
- Falls KEIN Edge → Seite NICHT erstellen.

## Gate 3: Minimum Content
- [ ] Mindestens 500 Wörter einzigartiger Content (nicht von anderen eigenen Seiten kopiert)
- [ ] Mindestens 3 FAQ-Fragen mit echten Antworten
- [ ] Kein Filler-Content ("In diesem Artikel erfahren Sie...")
- Falls unter 500 Wörter unique → Seite NICHT erstellen oder mit anderer Seite zusammenlegen.

## Gate 4: Kannibalisierung
- [ ] Gibt es schon eine eigene Seite, die das gleiche Keyword bedient?
- [ ] Würde die neue Seite einer bestehenden Seite Traffic wegnehmen?
- Falls JA → Bestehende Seite erweitern statt neue erstellen.

## Gate 5: Umsetzbarkeit
- [ ] Kann die Seite in einem Arbeitstag fertig werden (inkl. SEO, Schema, internes Linking)?
- [ ] Ist der Content zeitlos genug, dass er nicht monatlich aktualisiert werden muss?
- Falls die Seite mehr Pflege braucht als sie bringt → Seite NICHT erstellen.

## Gate 6: Trust-Check (bei Kosten/Empfehlungen)
- [ ] Alle Preise mit "Richtwert" oder "ca." gekennzeichnet?
- [ ] Quellen benennbar (auch wenn nicht verlinkt)?
- [ ] Disclaimer vorhanden wenn nötig?
- Falls Kosten/Empfehlungen ohne Grundlage → Seite NICHT erstellen bis Quellen da sind.

---

## Dokumentation

Wenn eine Seite das Kill-Gate besteht, kurzen Kommentar am Seitenanfang einfügen:

```html
<!-- Kill-Gate: 2026-03-30 | Edge: Tool (Planer-Integration) + Praxis (echte Kosten) | Autor: Bolle -->
```

## Ergebnis-Matrix

| Ergebnis | Aktion |
|---|---|
| Alle 6 Gates bestanden | Seite erstellen |
| Gate 1 oder 2 nicht bestanden | Seite NICHT erstellen |
| Gate 3 nicht bestanden | Content aufbauen, dann erneut prüfen |
| Gate 4 nicht bestanden | Bestehende Seite erweitern |
| Gate 5 oder 6 nicht bestanden | Vorarbeit leisten, dann erneut prüfen |
